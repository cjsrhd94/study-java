## 1. 클래스와 객체

### 1-1 클래스와 객체의 정의와 용도
 - 클래스의 정의: 객체를 정의해놓은 것, 객체의 설계도 또는 틀.
 - 클래스의 용도: 객체를 생성하는데 사용된다.
  

 - 객체의 정의: 실제로 존재하는 것, 사물 또는 개념.
 - 객체의 용도: 객체가 가지고 있는 기능과 속성에 따라 다르다.

### 1-2 객체와 인스턴스
 - 클래스로부터 만들어진 객체를 그 클래스의 인스턴스(instance)라고 한다.
 - 객체와 인스턴스는 같은 의미로 사용된다.

### 1-3 객체의 구성요소
 - 객체는 속성과 기능으로 구성된다.
    - 속성(property): 멤버변수(member variable) ,특성(attribute) ,필드(field), 상태(state)  
    - 기능(function): 메서드(method), 함수(function), 행위(behavior)

### 1-4 인스턴스의 생성과 사용
```java
Tv t;           //참조변수 t선언
t = new Tv();   //클래스의 객체를 생성하고, 생성된 인스턴스의 주소를 t에 저장
```

 - 인스턴스는 참조변수를 통해서만 다룰 수 있으며, 참조변수의 타입은 인스턴스의 타입과 일치해야 한다.
 - 참조변수에는 하나의 값(주소)만이 저장될 수 있으므로, 둘 이상의 참조변수가 하나의 인스턴스를 가리킬(참조할) 수 있다.
 - 그러나 하나의 참조변수로 여러개의 인스턴스를 가리키는 것은 불가능하다.


## 2. 변수와 메서드
### 2-1 선언 위치에 따른 변수의 종류
1. 클래스 변수  
    - 멤버변수중 `static`이 붙은 것.
    - 모든 인스턴스가 공통된 저장공간(변수)을 공유하게 된다.
    - 한 클래스의 모든 인스턴스들이 공통적인 값을 유지해야하는 속성의 경우 사용한다.
    - 인스턴스를 생성하지 않고 바로 사용할 수 있다.
    - 클래스 영역에 선언되며, 클래스가 메모리에 로딩 될 때 생성되어 프로그램이 종료될 때 까지 유지된다.
    - `public`을 앞에 붙이면 프로그램 내 어디서나 접근할 수 있는 전역변수(global variable)의 성격을 갖는다.
   

2. 인스턴스 변수
    - 멤버변수중 `static`이 붙지 않은 것.
    - 클래스 영억에 선언되며, 클래스의 인스턴스를 생성할 때 만들어진다.
    - 독립적인 저장공간을 가지기 때문에 서로 다른 값을 가질 수 있다.
    - 인스턴스마다 고유한 상태를 유지해야하는 속성의 경우 인스턴스변수로 선언한다.
   

3. 지역변수
    - 멤버변수를 제외한 모든 변수.
    - 메서드 내에서 선언되며, 변수 선언문이 수행될 때 생성된다.
    - 메서드가 종료되면 소멸되어 사용할 수 없다.

### 2-2 메서드(method)
- 메서드: 특정 작업을 수행하는 일련의 문장들을 하나로 묶은 것
```java
int add (int x, int y) {    //반환타입 메서드이름 (매개변수 선언)
    int result = x + y;
    
    return result;
        }
```
- 일반적인 변수선언과 달리 두 변수의 타입이 같아도 변수의 타입을 생략할 수 없다.
- 매개변수 선언에서 값을 입력받을 필요가 없다면 괄호안을 비워도 된다. 또한 입력해야할 개수가 많으면 배열이나 참조변수를 사용하면 된다.
  

- 반환값이 없는 경우 반환타입으로 `void`를 적어야 한다.
- `return`문은 단 하나의 값만 반환할 수 있으며, 반환타입과 일치하거나 자동 형변환이 가능한 것이어야 한다.
- `return`문의 반환값으로 수식의 결과값을 반환할 수도 있다.


### 2-3 JVM의 메모리 구조

1. 메서드 영역(method area)
    - 클래스가 사용되면 해당 클래스의 파일을 읽고 분석하여 클래스 데이터를 저장한다.
    - 이 때 클래스 변수도 이 영역에 함께 생성된다.
   

2. 힙(heap)
    - 인스턴스 즉 '인스턴스 변수들' 이 생성되는 공간.
   

3. 호출스택(call stack 또는 execution stack)
    - 메서드의 작업에 필요한 메모리 공간을 제공한다.
    - 메서드가 호출되면 메서드를 위한 메모리를 할당하며, 이 메모리는 지역변수들과 연산의 중간결과 등을 저장하는데 사용된다.
    - 메서드가 작업을 마치면 해당 메모리 공간은 반환되어 비워진다.

### 2-4 기본형 매개변수와 참조형 매개변수
- 기본형 매개변수: 변수의 값을 읽기만 할 수 있다. ex) 'x'
- 참조형 매개변수: 값이 저장된 곳의 주소를 알 수 있기 때문에, 변수의 값을 읽고 변경 할 수 있다. ex) 'd.x'


- 매개변수 뿐만 아니라 반환타입도 참조형이 될 수 있다.
- 모든 참조형 타입의 값은 `'객체의 주소'` 이므로 정수값(주소값)이 반환된다.

### 2-5 클래스 메서드(static메서드)와 인스턴스 메서드
- 클래스 메서드: 메서드 앞에 `static`이 붙는다. 클래스이름.메서드이름(매개변수)로 호출 가능
- 인스턴스 메서드: 메서드 앞에 `static`이 없다. 반드시 객체 생성 후 호출 가능


- 클래스의 멤버변수 중 모든 인스턴스에 공통된 값을 유지해야 하는 것이 있는지 살펴보고, 있으면 `static`을 붙인다.
- 작성한 메서드 중에서 인스턴스 변수나 인스턴스 메서드를 사용하지 않는 메서드에 `static`을 고려한다.

## 3. 오버로딩
- 오버로딩: 한 클래스 내에 같은 이름의 메서드를 여러 개 정의하는 것
- 메서드의 이름을 보고 기능을 쉽게 예측할 수 있게 해준다. (때문에 오류 가능성이 줄어든다.)

### 3-1 오버로딩의 조건
1. 메서드 이름이 같아야 한다.  
2. 매개변수의 개수 또는 타입이 달라야 한다.  
3. 반환타입은 오버로딩 구현과 아무런 연관이 없다.

### 3-2 가변인자(varargs)와 오버로딩
 - 가변인자: 메서드의 매개변수를 동적으로 지정할 수 있게 해주는 기능
 - '타입... 변수명' 으로 선언
 - 가변인자 외에 매개변수가 더있다면, 가변인자를 매개변수 중에서 가장 끝에 선언해야한다.
```java
String concatenate(String s1, String s2){ ... }
String concatenate(String s1, String s2, String s3){ ... }
String concatenate(String s1, String s2, String s3, String s4){ ... }

String concatenate(String... str){ ... }
```

## 4. 생성자(Constructor)
 - 생성자: 인스턴스가 생성될 때 호출되는 인스턴스 초기화 메서드


 - 생성자의 이름은 클래스 이름과 같아야 한다.
 - 생성자는 리턴 값이 없다.
 - 현재 사용하고 있는 인스턴스와 같은 상태를 갖는 인스턴스(모든 인스턴수 변수가 동일)를 더 만들 때 생성자를 이용한다.

### 4-1 매개변수가 있는 생성자와 기본 생성자
 - 생성자도 메서드처럼 매개변수를 선언하여 호출 시 값을 넘겨받아 인스턴스의 초기화 작업에 사용할 수 있다. 
 - 클래스 내에 생성자가 하나도 정의되지 않은 경우 컴파일러가 `'클래스이름() { }'`이라는 기본 생성자를 자동으로 추가해준다.
```java
class Car {
    String color;
    String gearType;
    int door;
    
    Car(){}                             //2. 정의된 생성자가 없기 때문에 기본 생성자가 들어가야 한다.
                                        
    Car(String c, String g, int d){     //1. 파라미터 값이 들어가있지 않다. 따라서 생성자가 정의되지 않았다.
        color = c;
        gearType = g;
        door = d;
    }
}
```
### 4-2 생성자에서 다른 생성자 호출하기
 - 두가지 조건을 만족시 생성자간에도 서로 호출할 수 있다.
   1. 생성자의 이름으로 클래스이름 대신 `'this(매개변수)'`를 사용한다.
   2. 한 생성자에서 다른 생성자를 호출할 때는 반드시 첫 줄에서만 호출이 가능하다.
```java
Car(String color){
    this(color,"auto", 4);
    door = 5;
}
```

 - `this`는 인스턴스 자신을 가리키는 참조변수로 인스턴스의 주소가 저장되어 있다.  
 - `this`는 모든 인스턴스에 숨겨진 지역변수로 존재하며 인스턴스 멤버만 사용할 수 있다.
```java
Car(String color, String gearType, int door) {
    this.color = color;         //this.color는 인스턴스변수, color은 생성자의 매개변수이다.
    this.gearType = gearType;
    this.door = door;
}
```
 - `this`와 `this(매개변수)`는 완전히 다르다!

## 5. 변수의 초기화

- 멤버변수와 배열은 초기화하지 않아도 변수의 자료형에 맞는 기본값으로 초기화된다.
- 지역변수는 사용 전 반드시 초기화해야 한다.

### 5-1 명시적 초기화(explicit initialization)
- 변수를 선언과 동시에 초기화 하는 것을 명시적 초기화라고 한다.
```java
class  Car {
    int door = 4;               //기본형 변수의 초기화
    Engine e = new Engine();    //참조형 변수의 초기화
}
```********
### 5-2 초기화 블럭(initialization block)
```java
Class InitBlock{
    static {/*클래스 초기화 블럭은 인스턴스 초기화 블럭 앞에 static을 붙인다. */}
           {/*인스턴스 초기화 블럭은 클래스 내에 블럭을 만들고 코드를 작성한다.*/}
        }
```
- 클래스 초기화 블럭은 클래스가 메모리에 처음 로딩될 때 한번만 수행된다.
- 인스턴스 초기화 블럭은 생성자와 같이 인스턴스를 생성할 때 마다 수행된다.
- 생성자보다 인스턴스 초기화 블럭이 먼저 수행된다.
